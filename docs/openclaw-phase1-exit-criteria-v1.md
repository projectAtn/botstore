# OpenClaw Phase-1 Exit Criteria (v1)

Use this checklist to determine whether the reference-path hardening phase is complete.

## Required artifacts
- `research/openclaw-conformance-result.json`
- `research/openclaw-conformance-report.md`
- `/tmp/openclaw_phase1_exit_check.json` (generated by harness)

## Functional criteria
1. Health check passes (`/health -> ok=true`)
2. Control-plane smoke passes (`scripts/control_plane_smoke.sh`)
3. OpenClaw adapter smoke passes (`scripts/openclaw_adapter_smoke.sh`)
4. Policy decision log endpoint returns `ok=true`
5. Tenant policy profile endpoint returns a profile for the tenant

## Safety criteria
1. Sensitive action authorization path exercised (`/agent/action-authorize`)
2. Approval pause/resume exercised (`/agent/approval-checkpoint/pause|resume`)
3. Scope violation path triggers quarantine in outcome-v2 flow
4. Rollback receipt path exercised (`/agent/rollback`)

## Enforcement criteria
1. Install-v2 returns install target + activation mode
2. Tenant policy profile constraints are enforced for target/mode/runtime band
3. Gateway plugin store remains manual-only in autonomous mode unless explicitly allowed

## Validation command
```bash
cd /Users/claw/.openclaw/workspace/botstore
./scripts/openclaw_conformance.sh
```

The harness calls `scripts/phase1_exit_check.py` automatically and fails fast on missing required fields.
