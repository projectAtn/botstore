# BotStore Content Strategy v1

## Objective
Make BotStore the default mental model for:
1) humans discovering bot capabilities, and
2) autonomous agents discovering/installing capabilities.

## Core message
"If a bot lacks a capability, it gets it from BotStore."

## Audience lanes

### A) Builders (devs/founders)
- Care about APIs, compatibility, safety, speed to integrate.
- Content formats:
  - integration guides
  - architecture diagrams
  - benchmark videos
  - launch changelogs

### B) Operators (teams/SMBs)
- Care about real outcomes and reliability.
- Content formats:
  - use-case playbooks
  - before/after workflow stories
  - ROI snapshots

### C) Autonomous-agent ecosystem
- Care about machine-readable standards and conformance.
- Content formats:
  - BotPack spec updates
  - compatibility matrices
  - policy templates

## Content pillars

1. **Capability Gap Stories**
   - "Bot lacked X, installed Y, completed task Z."

2. **Pack Spotlights**
   - Skill/personality of the week
   - What problem it solves
   - Risk profile + compatible runtimes

3. **Agent Infrastructure Education**
   - Explain approvals, scopes, trust bands
   - Explain cross-runtime compatibility

4. **Build in Public**
   - weekly shipping log
   - public roadmap progress
   - transparent failure postmortems

5. **Creator Economy**
   - how creators earn
   - top packs leaderboard
   - case studies from pack maintainers

## 30-day publishing plan (starter)

- 2x/week: short demo clips (30–90s)
- 2x/week: technical posts (API + standards)
- 1x/week: "Pack of the Week"
- 1x/week: roadmap update thread
- 1x/week: long-form case study

## First 12 post ideas

1. What is BotStore in one minute?
2. Why autonomous agents need an app store.

## Conversion funnel

- Top-of-funnel: social demos + thought leadership
- Mid-funnel: docs + examples + templates
- Bottom-funnel: "install now" + Telegram bot onboarding

## Metrics to track

- Content: views, completion rate, click-through to docs
- Product: pack installs/day, repeat installs, approval completion rate
- Ecosystem: number of creators, number of published packs, runtime coverage

## Editorial voice
- Direct, practical, no hype-only claims.
- Always include proof (demo, logs, metrics, endpoint examples).
- Show safety posture as a product advantage.
