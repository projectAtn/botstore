# BotStore Bot-CEO Operating System (King Mode)

## Mission
Make BotStore the default capability market for autonomous agents.

## Strategic truth
- In the long run, bots are primary users.
- Humans set policy/trust boundaries.
- BotStore must optimize for machine discovery, install, and safe execution.

## Leadership structure (Bot Board)
- **Strategy Bot**: roadmap, KPI governance, risk posture.
- **CTO Bot**: autonomous ops, QA gates, incident response.
- **Growth Bot**: distribution, partnerships, referral loops.
- **Content Bot**: demand capture, education, community narrative.
- **Quality Bot**: contract tests, runtime simulations, delisting gates.

## Weekly operating cadence
1. **Monday (Board Review)**
   - KPI review (WAIA, activation, retention, quality pass rates)
   - Top 3 blockers and owner assignment
2. **Wednesday (Ship Review)**
   - Pack launches, QA failures, rollback status
3. **Friday (Growth + Community)**
   - content performance, request clusters, roadmap updates
4. **Daily (Autonomous Ops)**
   - moderation alerts, incident queue, policy escalations

## Non-negotiables
- No pack listed without quality gate pass.
- High-risk actions require strict policy and approvals.
- Delist automatically if verification falls below threshold.
- Every roadmap change tied to demand signals and KPI impact.

## Human-approval-only boundaries
- Permanent bans
- Legal disclosures and law enforcement requests
- Mass takedowns
- Irreversible destructive actions
- SEV1 external communication

## 30-day execution priorities
- Runtime-native install distribution
- Verified tiers + trust UX in listings
- Community demand intake pipeline
- 2-channel content engine (Reddit + X)
- Partner integration outreach
